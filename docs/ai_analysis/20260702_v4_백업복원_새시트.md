## 📌 TL;DR
- 스프레드시트가 통째로 사라졌을 때 백업 JSON으로 '새 빈 시트'에 데이터를 채워 넣는 복원 기능(옵션1, Drive 권한 추가 없음).
- 관리자가 새 빈 시트를 만들어 서비스계정에 편집자 공유 → 앱 복원 화면에 시트 ID + 백업 JSON 업로드 → 원본 데이터 탭만 재생성.
- 수식/피벗 탭은 복원 제외, 확인 입력('복원') 필수, 실행 전 현재 백업 유도.

## 동작 구조
- [restoreToSpreadsheet(targetId, sheets)](../../src/lib/google-sheets.ts): 대상 접근 확인(get) → 없는 데이터 탭 생성(addSheet) → 탭별 clear 후 USER_ENTERED로 백업 값 기록. 복원 대상 = 교인명부·헌금·지출·소속·직분·헌금분류·지출분류·관·예산. 그 외(수식/피벗) 탭은 skipped로 반환.
- [/api/restore](../../src/app/api/restore/route.ts): 입력 검증 + 접근 오류(404/403)를 사용자 안내 메시지로 변환. 접근 확인이 먼저라 권한 없으면 **아무것도 기록되지 않음**.
- [RestoreForm](../../src/components/settings/RestoreForm.tsx): 서비스계정 이메일 안내, 대상 ID(기본=현재 시트), JSON 업로드, '복원' 확인 입력, 결과(탭별 행수·skip) 표시.
- [settings/backup](../../src/app/settings/backup/page.tsx): 백업 + 복원 섹션 통합. 서비스계정 이메일·현재 시트 ID를 서버에서 전달.

## 사용 순서 (사용자)
1. 구글 드라이브에서 새 빈 스프레드시트 생성
2. 서비스계정 이메일(`...iam.gserviceaccount.com`)에 편집자로 공유
3. 새 시트 URL의 ID(/d/ ~ /edit)를 복원 화면에 입력
4. 백업 JSON 업로드 → '복원' 입력 → 실행
5. 새 시트에 복원했으면 `GOOGLE_SPREADSHEET_ID`를 새 ID로 바꾸고 재배포

## 라이브 검증 (비파괴 범위)
| 항목 | 결과 |
|------|------|
| 복원 페이지 | 200, '복원(새 시트로)' 섹션·서비스계정 이메일 노출 ✓ |
| API 검증 | 빈 body/backup 없음 → 400 안내 ✓ |
| 접근 가드 | 없는 대상 ID → 400 '찾을 수 없습니다' (get 선행 → 미기록) ✓ |
| tsc / build | 통과, /api/restore 라우트 컴파일 ✓ |

## ⚠️ 한계
- 실제 시트에 기록하는 전체 복원 라이브 테스트는 프로덕션 데이터 보호를 위해 생략(clear+update는 앱이 이미 쓰는 표준 API). 새 빈 시트로 실사용 시 검증 권장.
- 백업은 값(FORMATTED_VALUE) 스냅샷이라 원본 수식은 담기지 않음 → 수식/피벗 탭은 복원 제외(원본 데이터만 되돌리면 재계산).
- 복원 후 앱이 새 시트를 쓰려면 환경변수 교체·재배포 필요(런타임 자동 전환 아님).
