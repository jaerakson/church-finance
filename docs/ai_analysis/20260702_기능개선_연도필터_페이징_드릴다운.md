- "전체 연도"가 2026만 나오던 버그 수정 — YearFilter가 param을 삭제하지 않고 `year=all`을 명시 전송하도록 변경.
- 재정집계표 항목(주정헌금·사례비 등) 클릭 → 일자별 로우데이터 + 인라인 수정/삭제 드릴다운 페이지 신설.
- 상세/성도별/교인명부 목록에 검색 + 페이지네이션(재사용 `PaginatedTable`) 추가.

## ⚠️ 가정 및 한계
- 코드값 실시간 반영(직전 세션 작업)은 유지된 상태에서 이번 UI 기능을 추가.
- 대용량 페이징은 클라이언트 사이드(전 행 전송 후 브라우저 페이징) — 교회 데이터 규모(수천 건)에서 충분. 수만 건 이상이면 서버 페이징 전환 필요.
- 라이브 검증은 로컬 dev(3001) + 실제 시트 기준. 코드값 추가 테스트 항목은 추가→삭제로 원상복구함.

## 변경 내역

### ① 연도 필터 "전체" 버그
- 원인: [YearFilter.tsx](../../src/components/ui/YearFilter.tsx)가 '전체' 선택 시 `params.delete('year')` → 페이지가 "param 없음 = 현재연도 기본" 으로 해석해 2026으로 회귀.
- 수정: 항상 `params.set('year', year)` (all 포함). 페이지 연도 해석을 `year==='all' ? 'all' : (year || 기본연도)`로 통일(`/expense`·`/offering` 포함) → 드롭다운 표시와 실제 필터 일치.

### ② 재정집계표 드릴다운 (신규)
- 신규 라우트: [finance/summary/[kind]/[typeKey]/page.tsx](../../src/app/finance/summary/[kind]/[typeKey]/page.tsx)
- summary 항목명을 `/finance/summary/{income|expense}/{typeKey}?year=` 링크로 변환(연도 유지).
- 해당 코드값의 헌금/지출을 연도 필터해 **기존 인라인편집 리스트 재사용**: 수입→`OfferingHistoryList`, 지출→`ExpenseHistoryList` (✏️수정·🗑️삭제 그대로 동작, 저장 시 `router.refresh()`).

### ③ 검색 + 페이지네이션
- 신규 재사용 컴포넌트: [PaginatedTable.tsx](../../src/components/ui/PaginatedTable.tsx) — tanstack-table 기반 전역검색·정렬·페이지크기(25/50/100/200)·페이지이동.
- 적용: `finance/expenses`, `finance/offerings`, `finance/members`(화면표) → PaginatedTable로 교체(CSV 전량 다운로드는 유지). `members` 교인명부(MemberTable)에는 페이지네이션 컨트롤 추가.
