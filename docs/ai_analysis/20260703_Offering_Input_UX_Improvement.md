## 📌 TL;DR
- 헌금 입력 화면의 오늘의 헌금 내역 요약 구역에서, 특정 헌금 종류(예: `주정헌금 (20명)`)를 클릭하면 아래 상세 이력 리스트가 해당 종류만 보여주도록 **인터랙티브 요약 연동 필터 시스템**을 성공적으로 구축했습니다.
- 필터를 선택하면 요약 박스 항목에 강렬하고 가시적인 **선택 상태 스타일링(파란색 배지화)**이 적용되며, 필터 중일 때 세부 내역 영역에 명시적인 필터 상태 표기 및 **`전체보기 ✕` 해제 버튼**이 나타나 최상의 UX 흐름을 구현합니다.
- 오리지널 재정집계표(summary) 등 다른 모든 소스코드는 일절 침해받지 않고 완벽히 고립 가동 중입니다.

## 세부 구현 및 튜닝 내역

### ① 종류별 소계 행 클릭 시 토글 필터링 탑재
- **연동 메커니즘**: `OfferingInputClient.tsx` 내부에서 헌금 분류 키에 대응하는 `selectedFilterType` 상태를 신설하였습니다.
- **필터 제어**:
  - `byType` 요약 영역에서 특정 항목을 클릭하면 `selectedFilterType`에 해당 키가 토글 바인딩됩니다.
  - 리스트를 필터링할 때 `[...todayList].filter((o) => !selectedFilterType || o.typeKey === selectedFilterType)` 조건이 실시간 적용됩니다.

### ② 필터 활성화 상태 가시성 극대화
- **선택 행 하이라이트**: 필터링 조건과 동일한 요약 행은 **배경 파란색(`bg-blue-600`), 텍스트 흰색, 볼드체 및 음영 효과**가 자동으로 전량 적용되어, 사용자가 자신이 "어떤 조건으로 세부 내역을 좁혀서 보고 있는지" 직관적으로 인지할 수 있게 도왔습니다.
- **세부 내역 라벨 변경**: 헌금 내역의 서브타이틀이 `세부 내역 (주정헌금)` 형태로 동적으로 바뀌고, 옆에 전체로 돌아갈 수 있는 **`전체보기 ✕` 단추**가 노출되어 빠른 탐색 및 필터 전환을 지원합니다.
